# 阶段8：发布评审 (Release Review)

## 阶段概述

发布评审是项目生命周期的最终阶段，由多角色共同参与。本阶段的核心目标是全面评估产品发布就绪度，做出发布决策。

## 主导角色

**架构师 + 产品经理**（联合主导）

## 参与角色

- 架构师：架构合规性审查
- 产品经理：业务价值验收
- 测试专家：质量评估确认
- 独立开发者：技术实现说明
- UI 设计师：UI 还原度验收（如有）

## 入口条件

- [ ] 测试验证通过
- [ ] 所有 P0/P1 缺陷已修复
- [ ] 质量评估报告已生成

## 上游依赖

- 阶段1 交付物：PRD 文档、验收标准
- 阶段2 交付物：架构设计文档
- 阶段7 交付物：测试报告、质量评估、发布建议

## 本阶段交付物

| 交付物 | 负责角色 | 格式要求 | 完成标准 |
|--------|---------|---------|---------|
| 发布评审报告 | 架构师 + 产品经理 | Markdown | 多维度评审结果 |
| 发布决策 | 架构师 + 产品经理 | Markdown | 通过/有条件通过/不通过 |
| 发布清单 | 独立开发者 | Markdown | 部署步骤和回滚方案 |
| 已知问题清单 | 测试专家 | Markdown | 未修复缺陷和影响 |
| 运维手册 | 独立开发者 | Markdown | 监控和告警配置 |

## 阶段执行指引

### 步骤1：架构合规性审查（架构师）
- 检查实现是否符合架构设计
- 评估技术债务
- 评估发布风险

### 步骤2：业务价值验收（产品经理）
- 确认所有验收标准已满足
- UAT 验收测试
- 评估用户影响

### 步骤3：质量评估确认（测试专家）
- 确认测试覆盖率达标
- 确认无高危缺陷
- 确认性能和安全指标

### 步骤4：UI 还原度验收（UI 设计师，如有）
- 对比设计稿和实现
- 评估视觉还原度
- 评估交互还原度

### 步骤5：发布决策
- 汇总所有评审结果
- 做出发布决策
- 制定发布计划

## 出口条件（门禁）

- [ ] 架构合规性审查通过
- [ ] 业务价值验收通过
- [ ] 质量评估确认通过
- [ ] 发布决策已做出
- [ ] 发布清单已准备
- [ ] 回滚方案已制定

## 发布决策标准

### ✅ 通过
- 所有门禁通过
- 无 P0 缺陷
- P1 缺陷有明确修复计划
- 性能和安全指标达标

### ⚠️ 有条件通过
- 大部分门禁通过
- 有少量 P1 缺陷，影响可控
- 有明确的修复时间表
- 发布后有密切监控计划

### ❌ 不通过
- 关键门禁未通过
- 有 P0 缺陷未修复
- 性能或安全指标不达标
- 需要返回对应阶段修复

## 常见风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 发布决策分歧 | 延期 | 多角色共识机制 |
| 回滚方案不完善 | 线上故障 | 强制回滚演练 |
| 监控不足 | 问题发现延迟 | 完善监控告警 |
| 文档不全 | 运维困难 | 强制运维手册 |
