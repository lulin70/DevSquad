# 阶段3：UI 设计 (UI Design)

## 阶段概述

UI 设计是项目生命周期的第三阶段，由 UI 设计师主导。本阶段的核心目标是将产品需求和架构约束转化为独特、生产级的界面设计。

## 主导角色

**UI 设计师** (UI Designer)

## 参与角色

- 产品经理：确认设计符合用户体验要求
- 架构师：确认设计符合架构约束
- 独立开发者：确认设计可实现

## 入口条件

- [ ] 架构设计评审通过
- [ ] 前后端接口契约已定义
- [ ] 技术栈和框架限制已明确

## 上游依赖

- 阶段1 交付物：PRD 文档、用户故事、验收标准
- 阶段2 交付物：架构设计文档、接口定义、技术约束

## 本阶段交付物

| 交付物 | 负责角色 | 格式要求 | 完成标准 |
|--------|---------|---------|---------|
| 设计哲学文档 | UI 设计师 | Markdown | 设计理念明确 |
| 风格指南 | UI 设计师 | Markdown | 字体/色彩/间距/动画完整 |
| 高保真原型 | UI 设计师 | HTML/图片 | 所有关键页面 |
| 交互流程图 | UI 设计师 | Mermaid | 用户操作路径完整 |
| 组件规格文档 | UI 设计师 | Markdown | 尺寸/状态/变体完整 |
| 设计系统 | UI 设计师 | Markdown/代码 | 可复用组件库 |
| 响应式方案 | UI 设计师 | Markdown | 断点和布局方案 |

## 阶段执行指引

### 步骤1：设计准备
- 深入理解用户故事和使用场景
- 确认技术约束（框架/浏览器限制）
- 收集品牌规范（如有）

### 步骤2：设计探索
- 回答设计思维4个问题
- 探索视觉方向（关键词 + 参考案例）
- 确定设计哲学

### 步骤3：风格定义
- 选择字体组合（展示字体 + 功能字体）
- 定义色彩方案（主色 + 辅色 + 语义色）
- 定义间距系统
- 定义动画规范

### 步骤4：原型设计
- 设计关键页面高保真原型
- 定义交互流程
- 设计组件规格
- 处理响应式方案

### 步骤5：设计评审
- 组织产品经理和独立开发者参与评审
- 确认设计覆盖所有用户故事
- 确认设计可实现

## 出口条件（门禁）

- [ ] 设计覆盖所有用户故事的核心流程
- [ ] 风格指南完整（字体/色彩/间距/动画）
- [ ] 高保真原型包含所有关键页面
- [ ] 交互流程清晰无断点
- [ ] 设计符合技术约束
- [ ] UI 设计评审通过（至少产品经理 + 独立开发者签字）

## 下游交接

交接给：**阶段4 - 测试设计（测试专家）** 和 **阶段5 - 任务分解（独立开发者）**
交接物：高保真原型、风格指南、组件规格、交互流程图

## 常见风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| AI slop 美学 | 设计平庸 | 遵循美学指南，避免通用方案 |
| 设计不可实现 | 开发困难 | 早期与开发者沟通技术约束 |
| 忽略深色模式 | 体验不一致 | 强制设计深色模式 |
| 无障碍缺失 | 用户受限 | WCAG 检查清单 |
