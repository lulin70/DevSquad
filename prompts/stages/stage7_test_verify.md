# 阶段7：测试验证 (Test Verification)

## 阶段概述

测试验证是项目生命周期的第七阶段，由测试专家主导。本阶段的核心目标是全面验证产品质量，确保产品满足所有验收标准。

## 主导角色

**测试专家** (Test Expert)

## 参与角色

- 独立开发者：修复缺陷
- 产品经理：UAT 验收
- 架构师：性能和安全评审

## 入口条件

- [ ] 开发实现完成
- [ ] 代码已通过代码审查
- [ ] 单元测试已通过

## 上游依赖

- 阶段4 交付物：测试策略、测试用例、质量门禁
- 阶段6 交付物：功能代码、单元测试、集成测试

## 本阶段交付物

| 交付物 | 负责角色 | 格式要求 | 完成标准 |
|--------|---------|---------|---------|
| 集成测试报告 | 测试专家 | Markdown | 所有集成场景结果 |
| 性能测试报告 | 测试专家 | Markdown | P50/P95/P99 指标 |
| 安全测试报告 | 测试专家 | Markdown | 漏洞清单和风险等级 |
| 缺陷清单 | 测试专家 | Markdown 表格 | 含优先级和复现步骤 |
| 质量评估报告 | 测试专家 | Markdown | 含质量评分 |
| 发布建议 | 测试专家 | Markdown | 通过/有条件通过/不通过 |

## 阶段执行指引

### 步骤1：集成测试
- 执行集成测试用例
- 验证模块间协作
- 记录测试结果

### 步骤2：性能测试
- 执行性能基准测试
- 验证性能指标满足验收标准
- 识别性能瓶颈

### 步骤3：安全测试
- 执行安全扫描
- 验证安全合规
- 识别安全漏洞

### 步骤4：缺陷管理
- 记录发现的缺陷
- 按严重程度分级
- 跟踪缺陷修复

### 步骤5：质量评估
- 汇总所有测试结果
- 计算质量评分
- 输出发布建议

## 出口条件（门禁）

- [ ] 所有 P0/P1 缺陷已修复并验证
- [ ] 测试覆盖率达标（单元 >80%，集成 >60%）
- [ ] 性能指标满足验收标准
- [ ] 安全测试无高危漏洞
- [ ] 测试报告评审通过

## 下游交接

交接给：**阶段8 - 发布评审（多角色）**
交接物：测试报告、缺陷清单、质量评估、发布建议

## 常见风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 严重缺陷 | 延期发布 | 优先修复 P0/P1 |
| 性能不达标 | 用户体验差 | 早期性能测试 |
| 安全漏洞 | 安全风险 | 安全扫描 + 审计 |
| 回归缺陷 | 质量下降 | 自动化回归测试 |
