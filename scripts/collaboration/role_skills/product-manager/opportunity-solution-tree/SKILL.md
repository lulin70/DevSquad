# Opportunity Solution Tree

## Purpose
Structure product discovery using the Opportunity Solution Tree (OST) framework by Teresa Torres. This ensures you explore the problem space before jumping to solutions.

## Context
Use this when exploring a new product area, conducting continuous discovery, or when the team is debating which solution to build without understanding the underlying opportunity.

## Instructions

1. **Define the Desired Outcome**
   - What business or customer metric are you trying to move?
   - Make it measurable (e.g., "Increase 7-day retention from 40% to 55%")
   - This goes at the top of the tree

2. **Identify Opportunities**
   - For each outcome, list 3-5 customer opportunities (pain points, unmet needs, desires)
   - Use customer interviews, support tickets, analytics data
   - Each opportunity should be framed from the customer's perspective
   - Ask: "What is the customer trying to accomplish but struggling with?"

3. **Generate Solutions for Each Opportunity**
   - For each opportunity, brainstorm 2-3 potential solutions
   - Don't evaluate yet — diverge first
   - Consider: new features, workflow changes, content, partnerships
   - Each solution should directly address the opportunity above it

4. **Design Experiments for Each Solution**
   - For each solution, identify the riskiest assumption
   - Design a quick experiment to test that assumption
   - Use the smallest possible test (fake door, prototype, interview)
   - Define clear success criteria before running the experiment

5. **Visualize the Tree**
   - Present as a structured tree:
     ```
     Desired Outcome
     ├── Opportunity 1
     │   ├── Solution 1A → Experiment 1A-1
     │   └── Solution 1B → Experiment 1B-1
     ├── Opportunity 2
     │   ├── Solution 2A → Experiment 2A-1
     │   └── Solution 2B
     └── Opportunity 3
         └── Solution 3A
     ```

6. **Prioritize Branches**
   - Which opportunity has the largest potential impact?
   - Which solution has the most uncertainty (and thus most to learn)?
   - Start with the branch that maximizes learning per unit of effort

## Notes
- The tree is a living document — update it as you learn
- Don't skip the opportunity layer — it's the most common mistake
- If you have more than 5 opportunities, you may need to narrow your outcome
- If you have only 1 solution per opportunity, you're not diverging enough
- Inspired by Teresa Torres' "Continuous Discovery Habits"
