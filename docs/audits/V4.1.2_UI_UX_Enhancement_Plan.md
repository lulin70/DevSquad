## 1. 背景与目标

### 1.1 现状摘要

DevSquad 当前 UI/UX 资产：
- **Streamlit Dashboard** (`scripts/dashboard/` 8 模块)：7 页面 + Morandi 配色 + 自定义 CSS
- **CLI 可视化** (`scripts/cli/cli_visual.py`)：ANSI 颜色 + Unicode 图标
- **FastAPI Docs** (`scripts/api_server.py`)：标准 Swagger UI
- **DAG 可视化** (`scripts/dashboard/dag_views.py`)：Mermaid 静态图
- **QA 框架** (`scripts/qa/`)：4 维度审计 + OKLCH + 4pt grid + 46 deterministic rules

### 1.2 核心目标

1. **用户感知提升**：Dispatch 实时执行可视化让用户"看到"AI 团队协作过程
2. **专业感升级**：暗色模式 + SVG 图标 + Toast 通知
3. **效率提升**：命令面板 + 键盘快捷键
4. **保持硬约束**：Morandi 配色、WCAG AA、anti-pattern 检测、视觉回归测试全通过

### 1.3 设计原则（per user_profile）

- ✅ 配色继续使用 Morandi（不刺眼）
- ✅ 避免 emoji 滥用（角色图标 SVG 化）
- ✅ 避免过度设计（保持简单）
- ✅ 文档先行：每个 Wave 实施前先更新本文档
- ✅ 充分测试：所有 UI 改动通过 WCAG AA + 46 deterministic rules + 视觉回归

## 5. 风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| Streamlit 限制无法实现复杂动画 | 用 st.fragment + st.status 实现渐进显示 |
| Plotly 依赖增加 | 已在 `[api]` extras 中，确认是否需要单独 `[dashboard]` extras |
| 暗色模式 CSS 与亮色冲突 | 用 CSS 变量 + data-theme 属性切换 |
| SVG 图标增加 bundle size | 内联 SVG，不引入图标库 |
| 视觉回归测试 flaky | 用 PIL ImageChops 像素 diff + 阈值容忍 |

## 7. Wave 2 执行记录

### 7.1 W2-T1 暗色模式

**文件**: `scripts/dashboard/components.py`

**设计方案**:
- 使用 CSS 变量 + `data-theme` 属性切换亮/暗模式
- `st.toggle("Dark Mode")` 存储到 `st.session_state["dark_mode"]`
- 通过 `st.markdown("<script>...</script>", unsafe_allow_html=True)` 注入 JS 设置 `<html data-theme="dark">`

**Morandi 暗色配色**（保持品牌一致性，仅调亮度）:
| 变量 | 亮色 | 暗色 |
|------|------|------|
| `--bg-primary` | `#F5F3F0` (Morandi bg) | `#1F1F23` (深灰蓝) |
| `--bg-card` | `#FFFFFF` | `#2A2A30` |
| `--text-primary` | `#4A4A4A` | `#E8E6E3` |
| `--text-secondary` | `#666` | `#A8A6A3` |
| `--accent-primary` | `#7B9EA8` (Morandi primary) | `#9DB5C2` (Morandi info，暗色背景更亮) |
| `--accent-success` | `#8FA886` | `#A8C2A0` |
| `--accent-warning` | `#C9A87C` | `#DDB589` |
| `--accent-danger` | `#B58484` | `#C9A0A0` |

**避免过度设计**: 不引入 theme_manager 独立模块，直接在 `apply_custom_css()` 中用 CSS 变量即可。Streamlit 的 `st.toggle` 是内置组件，无新依赖。

### 7.2 W2-T2 SVG 角色图标

**文件**: `scripts/dashboard/components.py`

**设计方案**:

- SVG 使用 `width=16 height=16 stroke=currentColor`，颜色随 CSS 上下文自适应

**7 角色 SVG 图标**（Lucide 风格，单色 stroke）:

**避免 emoji 滥用** (per user_profile)：所有 dashboard 内部使用 `get_role_icon(role, "svg")` 获取 SVG，仅 `ROLE_ICONS` 字典保留 emoji 用于纯文本场景（日志/CLI）。

### 7.3 W2-T3 Toast 通知系统

**文件**: `scripts/dashboard/components.py`

**设计方案**:

- 4 个 level: info / success / warning / error，对应 Morandi 4 色
- 5 秒后自动消失（CSS animation + keyframes）
- 多 toast 堆叠（每个 toast 唯一 id）

**避免过度设计**: 不引入 streamlit-toast 等第三方库，纯 CSS + JS 实现。Toast 仅替代 `st.success/st.error` 在"瞬时通知"场景的使用，form validation 等仍用 streamlit 原生。
