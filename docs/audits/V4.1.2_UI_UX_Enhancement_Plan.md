## 1. 背景与目标

### 1.1 现状摘要

DevSquad 当前 UI/UX 资产：
- **Streamlit Dashboard** (`scripts/dashboard/` 8 模块)：7 页面 + Morandi 配色 + 自定义 CSS
- **CLI 可视化** (`scripts/cli/cli_visual.py`)：ANSI 颜色 + Unicode 图标
- **FastAPI Docs** (`scripts/api_server.py`)：标准 Swagger UI
- **DAG 可视化** (`scripts/dashboard/dag_views.py`)：Mermaid 静态图
- **QA 框架** (`scripts/qa/`)：4 维度审计 + OKLCH + 4pt grid + 46 deterministic rules

### 1.2 核心目标

1. **用户感知提升**：Dispatch 实时执行可视化让用户"看到"AI 团队协作过程
2. **专业感升级**：暗色模式 + SVG 图标 + Toast 通知
3. **效率提升**：命令面板 + 键盘快捷键
4. **保持硬约束**：Morandi 配色、WCAG AA、anti-pattern 检测、视觉回归测试全通过

### 1.3 设计原则（per user_profile）

- ✅ 配色继续使用 Morandi（不刺眼）
- ✅ 避免 emoji 滥用（角色图标 SVG 化）
- ✅ 避免过度设计（保持简单）
- ✅ 文档先行：每个 Wave 实施前先更新本文档
- ✅ 充分测试：所有 UI 改动通过 WCAG AA + 46 deterministic rules + 视觉回归

## 5. 风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| Streamlit 限制无法实现复杂动画 | 用 st.fragment + st.status 实现渐进显示 |
| Plotly 依赖增加 | 已在 `[api]` extras 中，确认是否需要单独 `[dashboard]` extras |
| 暗色模式 CSS 与亮色冲突 | 用 CSS 变量 + data-theme 属性切换 |
| SVG 图标增加 bundle size | 内联 SVG，不引入图标库 |
| 视觉回归测试 flaky | 用 PIL ImageChops 像素 diff + 阈值容忍 |

## 6. Wave 1 执行记录

### 6.1 W1-T1 Dispatch 实时执行可视化

**文件**: `scripts/dashboard/dispatch_views.py`

**修改**:
- 替换 `st.spinner` 为 `st.status`（4 阶段渐进显示）
- 新增 `_predict_auto_roles()`：用 dispatcher.analyze_task() 预测角色，失败时回退到关键词启发式
- 新增 `_render_role_pipeline()`：彩色 badge 显示 7 角色并行执行状态
- 新增 `_ROLE_PIPELINE_ICONS` / `_ROLE_PIPELINE_COLORS`（Morandi 配色）

**4 阶段可视化**:
1. 📋 Phase 1/4: Initializing dispatcher and validating input
2. 🎯 Phase 2/4: Matching roles to task
3. ⚡ Phase 3/4: Agents executing in parallel
4. 🗳️ Phase 4/4: Building consensus and assembling report

### 6.2 W1-T2 性能图表交互化

**文件**: `scripts/dashboard/metrics_views.py`

**修改**:
- 新增 `_render_performance_charts()`：用 streamlit 内置 `st.bar_chart`（Altair 后端，hover 交互）
- 新增 `_MORANDI_CHART_COLORS`（6 色 Morandi 调色板）
- 新增 `_safe_float()`：处理 API 返回的 None/str/int 等多种类型
- 3 个图表区：
  1. Response Time Breakdown（Avg vs P95 柱状图）
  2. System Health Indicators（CPU/Mem/Success 柱状图）
  3. Throughput & Reliability（3 metric cards with deltas vs SLO）

**避免过度设计**: 不引入 plotly 重依赖，用 streamlit 内置能力即可

### 6.3 W1-T3 DAG 节点交互

**文件**: `scripts/dashboard/dag_views.py`

**修改**:
- 默认格式从 Mermaid 改为 `Graphviz (Interactive)`（st.graphviz_chart 是 streamlit 内置）
- 新增 `_build_interactive_dot()`：生成带 Morandi 状态色的 DOT 字符串
- 新增 `_render_graphviz_interactive()`：渲染图表 + 节点选择 selectbox + 详情面板
- 新增 `_GRAPHVIZ_STATUS_FILL` / `_GRAPHVIZ_STATUS_FONT`（6 状态色，Morandi 对齐）
- 节点详情面板显示：状态 badge / role / description / optional / order + 依赖 & 被依赖列表

**6 状态色映射**:
| 状态 | Fill | Font |
|------|------|------|
| pending | #F5F3F0 (Morandi bg) | #4A4A4A |
| running | #C9A87C (Morandi tan) | #FFFFFF |
| completed | #8FA886 (Morandi sage) | #FFFFFF |
| failed | #B58484 (Morandi rose) | #FFFFFF |
| skipped | #E0DDD8 (Morandi lt gray) | #888888 |
| blocked | #9B8AA4 (Morandi muted purple) | #FFFFFF |
