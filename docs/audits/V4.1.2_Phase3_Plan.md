  - **QA/Autonomous/Plugins** (4+3): V4.0.0 新增特性

**方案**：
```python
@dataclass
class DispatcherConfig:
    # Group 1: Feature flags (16)
    enable_warmup: bool = True
    enable_compression: bool = True
    # ...
    
    # Group 2: Paths (4)
    persist_dir: str | None = None
    # ...
    
    # Group 3: Injected components (8) — kept as Any for DI flexibility
    micro_task_planner: Any = None
    # ...
    
    # Backward-compat: kwargs 透传给 EnterpriseFeature
    extra: dict[str, Any] = field(default_factory=dict)


class MultiAgentDispatcher:
    def __init__(self, config: DispatcherConfig | None = None, **legacy_kwargs):
        self.config = config or DispatcherConfig.from_kwargs(**legacy_kwargs)
```

### 2.2 P1-12: ConfigManager Pydantic 强类型

**现状**：
- 无独立 `config_manager.py`/`config_loader.py` 文件
- `.devsquad.yaml` 152 行，混合 flat keys + nested `quality_control` 嵌套结构
- 散落引用：`PromptAssembler` 读 `quality_control`；其他模块读 flat keys
- 类型不安全：YAML 解析为 `dict`，无 schema 验证

**方案**：
```python
# scripts/collaboration/config_manager.py (新建)
from pydantic import BaseModel, Field
from pathlib import Path

class AIQualityControlConfig(BaseModel):
    enabled: bool = True
    hallucination_check: HallucinationCheckConfig = Field(default_factory=...)
    # ...

class ConsensusConfig(BaseModel):
    enabled: bool = True
    threshold: float = 0.7
    weights: dict[str, float] = Field(default_factory=...)
    # ...

class DevSquadConfig(BaseModel):
    quality_control: QualityControlConfig
    backend: str = "mock"
    base_url: str = ""
    timeout: int = 120
    # ... 所有 flat keys
    
    @classmethod
    def from_yaml(cls, path: Path) -> "DevSquadConfig":
        """Load and validate YAML config."""
        import yaml
        with open(path) as f:
            data = yaml.safe_load(f)
        return cls.model_validate(data)

class ConfigManager:
    """Singleton config manager with env var override."""
    def __init__(self, config_path: Path | None = None):
        self._config = DevSquadConfig.from_yaml(config_path or default_path())
    
    @property
    def config(self) -> DevSquadConfig:
        return self._config
```

**迁移策略**：
- 阶段 1: 新建 `config_manager.py`，与现有 dict-based 读取并行运行
- 阶段 2: 各模块逐步切换到 `ConfigManager.config.quality_control.consensus.threshold` 风格
- 阶段 3: 删除旧 dict-based 读取（V4.2.0）

**预计影响**：
- 代码改动：~300 行（新建 config_manager.py + Pydantic models）
- 测试改动：新增 ~200 行测试（config 加载/验证/环境变量覆盖）
- 风险：低（并行运行，渐进迁移）
