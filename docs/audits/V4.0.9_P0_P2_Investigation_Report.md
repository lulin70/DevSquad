## 二、P0 根因调查：5 个"预存测试失败"实为误报

### 2.1 误报项 #1: `test_lifecycle_command_in_cli_parser`

**之前会话报告**:
> `TypeError: unsupported operand type(s) for |: 'type' and 'NoneType'`
> 根因: `performance_fingerprint.py:63` 的 `str | None` 在 Python 3.9 报错

**实测复现**:
```bash
$ python -m pytest "tests/test_comprehensive_integration.py::TestCLIWithLifecycleIntegration::test_lifecycle_command_in_cli_parser" -v
tests/test_comprehensive_integration.py::TestCLIWithLifecycleIntegration::test_lifecycle_command_in_cli_parser PASSED [100%]
============================== 1 passed in 0.20s ==============================
```

**真实根因**:
1. **测试类名错误**: 之前会话使用 `TestLifecycleProtocolIntegration::test_lifecycle_command_in_cli_parser`，但该测试实际属于 `TestCLIWithLifecycleIntegration` 类。pytest 报 `no match`。
2. **Python 版本环境错误**: 之前会话在系统 Python 3.9 下运行测试。`performance_fingerprint.py:63` 使用 PEP 604 `str | None` 语法，需要 Python 3.10+。venv Python 3.12.13 下无此问题。
3. **结论**: 代码无 bug，测试无 bug。误报根因 = 错误环境 + 错误类名。

### 2.2 误报项 #2: `test_start_script.py` 3 个测试失败

**之前会话报告**:
> `✗ Python 3.9 检测到，需要 Python >= 3.10`
> 根因: `start.sh` 环境检查在 `--help` 之前执行 exit 1

**实测复现**:
```bash
$ python -m pytest tests/test_start_script.py -v
============================== 14 passed in 2.69s ==============================
```

**真实根因**:
1. **Python 版本环境错误**: 之前会话在系统 Python 3.9 下运行。`start.sh` L34 用 `python3` 拾取系统 3.9，触发 L38-41 的版本检查 `exit 1`。
2. **venv 环境下无问题**: venv Python 3.12.13 下 `python3` 解析为 venv python，版本检查通过，`--help` 正常到达。
3. **结论**: `start.sh` 设计正确（环境检查在前是合理的运维约束）。误报根因 = 错误环境。

### 2.3 误报项 #3: `test_real_llm_smoke.py` Moka 测试失败

**之前会话报告**:
> 路径错误: `file or directory not found: tests/test_real_llm_smoke.py`
> 实际位置: `tests/smoke/test_real_llm_smoke.py`

**实测复现（修正路径后）**:
```bash
$ python -m pytest tests/smoke/test_real_llm_smoke.py -v --timeout=90
tests/smoke/test_real_llm_smoke.py::TestMokaLLMSmoke::test_dispatch_with_moka_llm PASSED [ 66%]
tests/smoke/test_real_llm_smoke.py::TestMokaLLMSmoke::test_dispatch_multi_role_moka PASSED [ 83%]
tests/smoke/test_real_llm_smoke.py::TestMokaLLMSmoke::test_moka_result_contains_findings PASSED [100%]
=================== 3 passed, 3 skipped in 149.14s ===================
```

**真实根因**:
1. **路径错误**: 之前会话用 `tests/test_real_llm_smoke.py`，实际在 `tests/smoke/` 子目录。
2. **瞬时网络抖动**: 首次运行时报 `openai.APIConnectionError: Connection error. [Errno 61] Connection refused`。直接 curl `https://api.moka-ai.com/v1/chat/completions` 可达（HTTP 404 GET，预期）。重新运行后 3 个 Moka 测试全部 PASSED。
3. **结论**: 代码无 bug。误报根因 = 错误路径 + 瞬时网络抖动。

### 2.4 误报根因汇总

| 误报项 | 之前报告根因 | 真实根因 | 代码是否需修改 |
|--------|-------------|----------|---------------|
| #1 lifecycle_command_in_cli_parser | `str | None` 语法 | 错误环境(Python 3.9) + 错误类名 | 否 |
| #2 test_start_script 3 tests | start.sh 环境检查顺序 | 错误环境(Python 3.9) | 否 |
| #3 test_real_llm_smoke Moka | - | 错误路径 + 瞬时网络抖动 | 否 |

**教训 (Lessons Learned)**:
- 测试失败排查必须先确认运行环境（`python --version` + `which python`），不能默认环境正确
- 测试路径和类名必须用 `pytest --co` 收集确认，不能凭记忆
- 瞬时网络错误应重试 2-3 次再下结论
- 已记录到 `project_memory.md` 防止重复犯错
