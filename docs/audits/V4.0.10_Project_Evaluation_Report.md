| P2 | 部分模块行数较大（dispatch_steps.py 等） | 多文件 | ⚠️ 非 God Class（SRP 分析通过） |

### 修复详情

**P1-1: redis_url 凭据泄露修复**

新增 `_mask_redis_url()` 函数，在所有暴露 redis_url 的位置进行脱敏：
```python
def _mask_redis_url(url: str) -> str:
    """redis://:secret@host:6379/0 → redis://***@host:6379/0"""
    parsed = urlparse(url)
    if parsed.password:
        # 替换密码为 ***，保留 host:port/db
        ...
```

覆盖 5 个暴露点：`__init__` 日志、`_get_client` 日志、`stats()` 返回、`health_check()` 返回、`__repr__`。

**P1-2: health_check() RedisConnectionError 捕获修复**

```python
# 修复前
except (ConnectionError, TimeoutError, OSError, RuntimeError) as e:

# 修复后
except (RedisConnectionError, ConnectionError, TimeoutError, OSError, RuntimeError) as e:
```

这是 P1-A redis_cache.py bug 修复（v4.0.10）的延续——`RedisConnectionError` 继承自 `Exception` 而非 `builtins.ConnectionError`，`health_check()` 的 except 子句未捕获它，导致连接失败时抛出未捕获异常而非返回 "unhealthy" 状态。
