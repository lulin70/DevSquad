## 6. 活文档维护机制（长期保障）

### 6.1 现有 CI 保障

| 检查项 | 脚本 | 作用 |
|--------|------|------|
| 版本一致性 | `scripts/check_version_consistency.py` | 15 处版本号一致性检查 |
| 文档一致性 | `scripts/check_doc_consistency.sh` | 必需文档是否引用当前 VERSION |
| 依赖同步 | `scripts/check_dependency_sync.py` | requirements-dev.txt 与 pyproject.toml [dev] 一致性 |

### 6.2 新增保障建议

| 检查项 | 方式 | 作用 |
|--------|------|------|
| 模块数一致性 | CI 中 grep "149+\|48 modules\|173+" 检查是否残留旧模块数 | 防止模块数回退 |
| SUPERSEDED 审计 | 定期检查是否有文档被标记 SUPERSEDED 超过 30 天未处理 | 防止 SUPERSEDED 成为永久状态 |
| 文档更新触发器 | PR 中如修改 scripts/collaboration/*.py，提醒检查 SKILL.md 模块表 | 代码变更触发文档审查 |

### 6.3 文档维护责任矩阵

| 文档 | 主责角色 | 更新触发条件 |
|------|---------|------------|
| SKILL.md | Architect | 新增/删除模块时 |
| ARCHITECTURE_V4.md | Architect | 架构变更时 |
| SPEC.md | PM + Architect | 版本发布时 |
| CONSTITUTION.md | PM | 生命周期/权限/安全模型变更时 |
