用户指出 "UI/UX 设计是 DevSquad 比较薄弱的环节"，要求评估 GitHub 上的 taste-skill 和 impeccable 两个开源 UI/UX Skill，吸收借鉴融合到 DevSquad，增强 UI/UX 能力。

| **VisualRegressionChecker** | `scripts/qa/visual_regression.py` | 179 | PIL ImageChops 像素级 Diff + 区域检测 | N/A |

**关键发现**：
1. `PromptDials` 已在 docstring 中明确 "Inspired by Leonxlnx/taste-skill's VARIANCE/MOTION/DENSITY dials" — DevSquad 已借鉴过 taste-skill 的 Dials 概念，但只在 **prompt 层面**，未延伸到 **视觉设计层面**
2. `UIUXAnalyzer` 已有 4 维度审计框架，但规则库仅 ~10 条，远少于 impeccable 的 46 条确定性规则
3. `UETestFramework` 已有完整的启发式评估框架，但缺乏 **设计准则（design pillars）** 视角
4. `VisualRegressionChecker` 是像素级 diff，缺乏 **语义级设计审查**

## 二、taste-skill 详细分析

### 2.1 核心概念

#### 2.1.1 Taste Dials（品味旋钮）

三个 0-1 范围的参数化品味控制：

| Dial | 范围 | 含义 | 低值 | 高值 |
|------|------|------|------|------|
| DESIGN_VARIANCE | 0-1 | 设计变化度 | 统一/克制 | 多样/丰富 |
| MOTION_INTENSITY | 0-1 | 动效强度 | 静态/微妙 | 强烈/丰富 |
| VISUAL_DENSITY | 0-1 | 视觉密度 | 留白/简约 | 紧凑/信息密集 |

**与 DevSquad PromptDials 对比**：

| 维度 | DevSquad PromptDials | taste-skill Taste Dials |
|------|---------------------|------------------------|
| **作用域** | Prompt 文本调优 | 视觉设计调优 |
| **参数** | VERBOSITY/CREATIVITY/RISK_TOLERANCE (1-5) | DESIGN_VARIANCE/MOTION_INTENSITY/VISUAL_DENSITY (0-1) |
| **应用** | Worker prompt 注入 | UIUXAnalyzer 审计阈值调整 |
| **已有** | ✅ 已实现 (V3.9.0) | ❌ 未实现 |

#### 2.1.2 Anti-pattern Bans（反模式禁令）

taste-skill 明确禁止的 AI 前端反模式：

| 反模式 | 说明 | DevSquad 现状 |
|--------|------|--------------|
| border-left accent stripes | 左边框强调条 | ❌ 未检测 |
| gradient text | 渐变文字 | ❌ 未检测 |
| glassmorphism overuse | 毛玻璃过度使用 | ❌ 未检测 |
| Inter/Roboto/DM Sans fonts | 过度使用字体 | ❌ 未检测 |
| purple-to-blue gradients | 紫蓝渐变 | ❌ 未检测 |
| cards nested in cards | 卡片嵌套 | ❌ 未检测 |

#### 2.1.3 6 Meta-skills（元技能分层）

| 元技能 | 定位 | DevSquad 借鉴价值 |
|--------|------|------------------|
| taste-skill (flagship) | 旗舰技能 | 学习其 brief inference 机制 |
| redesign-skill | 审计+修复 | 与 RedesignAuditor 互补 |
| soft-skill | 昂贵过滤器 | 学习其成本意识 |
| output-skill | taskmaster | N/A（DevSquad 有自己的 taskmaster） |
| minimalist-skill | Linear/Notion 风格 | 学习其风格预设机制 |
| brutalist-skill | CRT 美学 | N/A（DevSquad 不做具体 UI） |

### 2.2 技术细节

- **OKLCH 色彩空间**: 替代 sRGB，更感知均匀
- **4pt 网格间距**: 语义化间距尺度
- **禁止 width/height 动画**: 只用 transform
- **禁止 bounce easing**: 物理不准确
- **v2 experimental**: brief inference（简短推断）、design-system map、硬性 em-dash 禁令、canonical GSAP 代码骨架、redesign-audit 协议、严格 pre-flight check

## 三、impeccable 详细分析

### 3.1 核心概念

#### 3.1.1 7 Design Pillars（设计支柱）

| Pillar | 说明 | DevSquad 现状 |
|--------|------|--------------|
| Typography | 排版（字体/行高/字距） | ❌ 未检测 |
| Color | 色彩（对比/和谐/可访问性） | ✅ WCAG 对比度 + HSV 刺眼配色 |
| Spatial design | 空间设计（间距/对齐/网格） | ❌ 未检测 |
| Responsiveness | 响应式（断点/流式布局） | ✅ 视口溢出检测 |
| Interactions | 交互（反馈/状态/手势） | ✅ 按钮尺寸/focus |
| Motion | 动效（过渡/缓动/性能） | ❌ 未检测 |
| UX writing | UX 文案（语气/清晰/简洁） | ❌ 未检测 |

**DevSquad 覆盖 3/7 支柱，缺失 4/7**。

#### 3.1.2 46 确定性检测规则（核心价值）

impeccable 的 46 条规则 **不需要 LLM/API key**，纯确定性检测，与 DevSquad Python 代码风格高度匹配。

规则分布（基于 7 pillars）：

| Pillar | 规则数（估） | 示例 |
|--------|-------------|------|
| Typography | ~8 | line-height < 1.2、font-size < 12px、字重组合不当 |
| Color | ~8 | 对比度不足、渐变文字、紫蓝渐变、灰字彩底 |
| Spatial | ~6 | 间距非 4pt 倍数、对齐不一致、负 margin |
| Responsiveness | ~6 | 固定宽度、无断点、水平溢出 |
| Interactions | ~8 | 按钮 < 44px、无 hover/focus、无 loading 状态 |
| Motion | ~5 | width/height 动画、bounce easing、动画 > 1s |
| UX writing | ~5 | 占位符不清、按钮文案模糊、错误消息技术化 |

**DevSquad 现有规则库 ~10 条，融合后目标 50+ 条**。

#### 3.1.3 23 Commands（命令词表）

| 类别 | 命令 | DevSquad 借鉴价值 |
|------|------|------------------|
| 初始化 | init / document / extract | 学习其上下文收集协议 |
| 创建 | craft / shape | 学习其分阶段创建流程 |
| 审查 | critique / audit | 与 UIUXAnalyzer 互补 |
| 优化 | polish / bolder / quieter / distill / harden | 学习其语气控制词表 |
| 专项 | onboard / animate / colorize / typeset / layout / delight | 学习其专项命令组织 |
| 模式 | live / overdrive / clarify / adapt / optimize | 学习其模式切换 |

**DevSquad CLI 已有 6 个 lifecycle shortcuts（spec/plan/build/test/review/ship），未来可参考 impeccable 的细分命令词表**。

#### 3.1.4 PRODUCT.md + DESIGN.md 上下文协议

impeccable 通过 `/init` 命令收集两个上下文文件：

| 文件 | 用途 | DevSquad 对应 |
|------|------|--------------|
| PRODUCT.md | 产品上下文（用户/目标/场景） | 部分由 PRD 覆盖 |
| DESIGN.md | 设计准则（风格/约束/偏好） | ❌ 缺失 |

**DevSquad 已有 SPEC.md（技术规范）和 GLOSSARY.md（术语表），但缺失 DESIGN.md（设计准则）**。

#### 3.1.5 Live Browser Iteration Mode

impeccable 支持 `live` 命令进入实时浏览器迭代模式，与设计稿实时对照。

**DevSquad 已有 VisualRegressionChecker（Playwright 软依赖）和 Dashboard，未来可增加实时巡检模式**。

## 五、融合方案详解

### 5.1 UI-P0-1: 46 确定性规则引擎（impeccable）

**借鉴来源**: pbakaus/impeccable — 46 deterministic detector rules
**落地模块**: `scripts/qa/uiux_analyzer.py` 新增 `DeterministicRuleEngine` 类
**Matt/impeccable 理念**:
- 46 条规则纯确定性检测，不需要 LLM/API key
- 基于 7 design pillars 组织
- 每条规则有明确的 pass/fail 判定逻辑

**DevSquad 融合**:
- 新增 `DeterministicRuleEngine` 类，管理 46+ 条规则
- 规则按 7 pillars 分类，映射到 DevSquad 4 维度（a11y/interaction/layout/ux_antipattern）
- 扩展规则库从 ~10 条到 50+ 条
- 规则格式：`Rule(id, pillar, severity, check_fn, fix_hint)`

**7 pillars → 4 维度映射**:

| impeccable Pillar | DevSquad 维度 | 新增规则数 |
|------------------|--------------|-----------|
| Typography | a11y + ux_antipattern | 8 |
| Color | a11y | 5（已有 3，新增 2） |
| Spatial design | layout | 6 |
| Responsiveness | layout | 4（已有 1，新增 3） |
| Interactions | interaction | 5（已有 2，新增 3） |
| Motion | ux_antipattern | 5 |
| UX writing | ux_antipattern | 5 |

**验收标准**: ≥46 条规则 + DeterministicRuleEngine 类 + ≥20 测试

### 5.2 UI-P0-2: Taste Dials 视觉版（taste-skill）

**借鉴来源**: Leonxlnx/taste-skill — Taste Dials (DESIGN_VARIANCE/MOTION_INTENSITY/VISUAL_DENSITY)
**落地模块**: `scripts/qa/taste_dials.py` 新建（区别于已有的 `scripts/collaboration/prompt_dials.py`）
**taste-skill 理念**:
- 三个 0-1 范围参数控制视觉品味
- DESIGN_VARIANCE: 统一↔多样
- MOTION_INTENSITY: 静态↔强烈
- VISUAL_DENSITY: 留白↔紧凑

**DevSquad 融合**:
- 新建 `TasteDials` 类（0-1 范围，区别于 PromptDials 的 1-5）
- UIUXAnalyzer 审计时读取 TasteDials 配置，动态调整阈值
  - VISUAL_DENSITY 高 → 放宽布局密度检测阈值
  - MOTION_INTENSITY 高 → 放宽动效规则
  - DESIGN_VARIANCE 高 → 放宽一致性检测
- 与 PromptDials 职责分离：PromptDials 管 prompt，TasteDials 管视觉

**验收标准**: TasteDials 类 + UIUXAnalyzer 集成 + ≥12 测试

### 5.3 UI-P0-3: PRODUCT.md + DESIGN.md 协议（impeccable）

**借鉴来源**: pbakaus/impeccable — PRODUCT.md + DESIGN.md context protocol
**落地模块**: `docs/spec/DESIGN.md` 新建 + `scripts/qa/uiux_analyzer.py` 加载逻辑
**impeccable 理念**:
- `/init` 命令收集产品上下文和设计准则
- 后续审查基于这些上下文，而非通用规则

**DevSquad 融合**:
- 新建 `docs/spec/DESIGN.md` — DevSquad 项目设计准则
  - 项目视觉风格（Morandi 色系，符合用户偏好）
  - 间距尺度（4pt 网格）
  - 字体策略
  - 动效约束
  - 响应式断点
- UIUXAnalyzer 新增 `load_design_context()` 方法，加载 DESIGN.md
- 审计时按 DESIGN.md 的项目特定准则检测，而非仅通用规则

**验收标准**: DESIGN.md 建立 + UIUXAnalyzer 加载 + ≥8 测试

### 5.4 UI-P1-1: Anti-pattern Bans 扩展（taste-skill）

**借鉴来源**: Leonxlnx/taste-skill — Anti-pattern Bans
**落地模块**: `scripts/qa/uiux_analyzer.py` 的 `_check_ux_antipattern` 扩展
**taste-skill 理念**:
- 6 类 AI 前端反模式必须禁止
- 每类有明确的 CSS 检测特征

**DevSquad 融合**:
- `_check_ux_antipattern` 新增 6 条规则：
  1. `border_left_accent_stripes` — 检测 `border-left: Npx solid <color>`
  2. `gradient_text` — 检测 `background-clip: text` + `gradient`
  3. `glassmorphism_overuse` — 检测 `backdrop-filter: blur()` 使用次数
  4. `overused_fonts` — 检测 Inter/Roboto/DM Sans 字体声明
  5. `purple_blue_gradient` — 检测紫蓝渐变背景
  6. `nested_cards` — 检测 `.card .card` 嵌套

**验收标准**: 6 条新规则 + ≥10 测试

### 5.5 UI-P1-2: 7 Design Pillars 词汇（impeccable）

**借鉴来源**: pbakaus/impeccable — 7 Design Pillars
**落地方式**: `docs/spec/GLOSSARY.md` 收录 + UETestFramework SKILL.md 更新
**impeccable 理念**:
- 7 个设计支柱作为 UI/UX 评估的词汇框架

**DevSquad 融合**:

**验收标准**: GLOSSARY 收录 + SKILL.md 更新

### 5.6 UI-P1-3: OKLCH 色彩空间检测（taste-skill）

**借鉴来源**: Leonxlnx/taste-skill — OKLCH colour spaces
**落地模块**: `scripts/qa/uiux_analyzer.py` 色彩检测增强
