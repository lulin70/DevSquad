## 1. 各角色评审意见

### 1.1 Architect (权重: 1.5)

**评分**: 8/10

**认可**:
- CodeKnowledgeGraph 是正确的架构方向，CodeMapGenerator 升级为持久化图谱解决了核心痛点
- 融合矩阵清晰，每个新模块都有明确的集成点
- Phase 1-4 实施顺序合理，图谱优先作为其他功能基础

**关注**:
- **A1 (P0)**: CodeKnowledgeGraph 的 SQLite 存储需要设计表结构。建议三张表：`symbols` (符号定义)、`edges` (调用/依赖关系)、`files` (文件元数据)。不要用 JSON blob 存储，否则查询性能不达标。
- **A2 (P1)**: PromptDials 与现有 3 变体的关系需要明确。建议保留 `variant` 参数作为快捷方式（`variant="concise"` 等价于 `dials=PromptDials(verbosity=1)`），而非删除。
- **A3 (P2)**: 多宿主适配层的 HostAdapter 协议需要定义 `uninstall_skill()` 方法，Ponytail 和 CodeGraph 都有卸载功能。

**结论**: ✅ Approve (附 3 条建议)

### 1.2 Product Manager (权重: 1.2)

**评分**: 9/10

**认可**:
- 痛点分析有数据支撑（CodeGraph 58% 工具调用减少）
- 验收标准可量化
- 向后兼容设计合理

**关注**:
- **P1**: V39-06 RBAC/Audit 集成应该提到 P0。生产就绪度是当前最大短板，RBAC 是企业客户的硬需求。

**结论**: ✅ Approve (附 1 条建议)

### 1.3 Security Expert (权重: 1.5)

**评分**: 7/10

**认可**:
- YagniChecker 的"偷懒但不疏忽"原则正确，安全/错误处理永不跳过
- Audit 日志复用 secret_patterns 过滤敏感数据

**关注**:
- **S1 (P0)**: CodeKnowledgeGraph 存储代码结构信息，如果被恶意查询可能泄露架构信息。需要在 MCP codegraph_explore 工具中加入权限检查——只有授权用户才能查询图谱。
- **S2 (P1)**: RBAC 集成需要明确权限模型。建议采用 RBAC0（用户-角色-权限三层），不要过度设计。权限粒度：role-level（哪些 roles 可用）+ mode-level（哪些 mode 可用）。
- **S3 (P1)**: Audit 日志需要防篡改。建议使用 append-only 模式 + SHA-256 链式哈希（类似 CheckpointManager）。

**结论**: ⚠️ Approve with Conditions (3 条安全要求必须满足)

### 1.4 Tester (权重: 1.0)

**评分**: 8/10

**认可**:
- 每个新模块 ≥15 测试的要求合理
- 验收标准中的性能指标可测试

**关注**:
- **T1 (P0)**: CodeKnowledgeGraph 需要测试增量更新的正确性——文件修改后图谱是否正确更新。建议增加"修改函数名→查询旧名返回空→查询新名返回正确"的测试。
- **T2 (P1)**: YagniChecker 的 80% 准确率需要定义测试集。建议创建 20 个标注好的微任务作为黄金测试集。
- **T3 (P1)**: PromptDials 的测试需要验证不同 dials 值确实产生不同的 prompt。建议用 prompt 差异度（编辑距离）量化。

**结论**: ✅ Approve (附 3 条测试建议)

### 1.5 Coder (权重: 1.0)

**评分**: 8/10

**认可**:
- 融合方案清晰，知道每个新模块该放在哪里
- 向后兼容方案合理

**关注**:
- **C1 (P1)**: CodeKnowledgeGraph 的查询接口应该返回 dataclass 而非 dict，方便类型检查。建议定义 `SymbolInfo`、`CallEdge`、`DependencyEdge` 等 dataclass。
- **C2 (P2)**: 多宿主适配层的 GenericAdapter 需要处理 SKILL.md frontmatter 格式差异。Trae 用 YAML frontmatter，`npx skills` 也用 YAML 但字段不同。

**结论**: ✅ Approve (附 2 条建议)

### 1.6 DevOps (权重: 0.8)

**评分**: 7/10

**认可**:
- CI/CD 已有 mypy（V3.8.1 新增），新模块可受益

**关注**:
- **D1 (P0)**: CodeKnowledgeGraph 的 SQLite 文件 (`codegraph.db`) 需要加入 `.gitignore`。大项目的图谱可能几十 MB，不应提交到仓库。
- **D2 (P1)**: 图谱构建可能需要较长时间（大项目），CI 中需要缓存图谱。建议在 CI 中使用 `codegraph init --if-absent` 模式。
- **D3 (P2)**: 多宿主适配层的安装/卸载需要在 CI 中测试。建议增加 `pytest tests/test_host_adapter.py` 到 CI。

**结论**: ✅ Approve (附 3 条建议)

### 1.7 UI Designer (权重: 0.5)

**评分**: 7/10

**认可**:
- 三维度调节对 UI 设计任务有帮助

**关注**:
- **U1 (P2)**: 多宿主适配层应考虑 Streamlit Dashboard 的集成——在 Dashboard 中可视化图谱结构和查询结果。

**结论**: ✅ Approve (附 1 条建议)
