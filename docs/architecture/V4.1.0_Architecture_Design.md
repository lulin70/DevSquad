## 一、设计目标

### 1.1 核心原则

1. **理念融合而非功能堆砌** — 每个借鉴项必须强化 DevSquad 7 角色协作
2. **模块边界清晰** — 28 项按架构层组织，无交叉污染
3. **向后兼容** — 所有借鉴是增强而非重写
4. **文档先行** — 四文档体系（GLOSSARY/ADR/DESIGN/SPEC）作为基础设施先建

### 1.2 架构约束

| 约束 | 说明 |
|------|------|
| 无新外部依赖 | 所有借鉴通过修改现有模块实现 |
| Python 3.10+ | 保持现有技术栈 |
| 纯确定性检测 | UI/UX 规则不需要 LLM |
| SRP 分离 | TasteDials(视觉) vs PromptDials(prompt) 职责分离 |

**三准则**（全满足才写 ADR）：
