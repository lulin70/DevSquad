## 1. V39-01: CodeKnowledgeGraph

### 1.1 模块结构

```
scripts/collaboration/
├── code_knowledge_graph.py    # CodeKnowledgeGraph 主类
├── code_graph_storage.py      # SQLite 存储层
├── code_graph_query.py        # 查询接口 (dataclass 返回)
└── code_map_generator.py      # (现有, 保留, 作为构建器)
```

### 1.2 数据模型 (dataclass)

```python
@dataclass
class SymbolInfo:
    name: str
    symbol_type: str  # function|class|method|module
    file_path: str
    line_start: int
    line_end: int
    docstring: str
    signature: str  # 函数签名/类定义

@dataclass
class CallEdge:
    caller: str  # 符号全名
    callee: str  # 符号全名
    file_path: str
    line: int

@dataclass
class DependencyEdge:
    source_module: str
    target_module: str
    import_type: str  # import|from_import|relative
```

### 1.3 SQLite 表结构 (采纳 A1 建议)

```sql
CREATE TABLE symbols (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    symbol_type TEXT NOT NULL,
    file_path TEXT NOT NULL,
    line_start INTEGER,
    line_end INTEGER,
    docstring TEXT,
    signature TEXT,
    updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_symbols_name ON symbols(name);
CREATE INDEX idx_symbols_file ON symbols(file_path);

CREATE TABLE edges (
    id INTEGER PRIMARY KEY,
    edge_type TEXT NOT NULL,  -- 'call'|'dependency'|'import'
    source TEXT NOT NULL,
    target TEXT NOT NULL,
    file_path TEXT,
    line INTEGER,
    updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_edges_source ON edges(source);
CREATE INDEX idx_edges_target ON edges(target);

CREATE TABLE files (
    path TEXT PRIMARY KEY,
    hash TEXT NOT NULL,
    line_count INTEGER,
    last_scanned TEXT DEFAULT CURRENT_TIMESTAMP
);
```

### 1.4 接口定义

```python
class CodeKnowledgeGraph:
    def __init__(self, db_path: Path): ...

    def build_from_project(self, project_root: Path) -> int:
        """Full build. Returns number of symbols indexed."""

    def update_file(self, file_path: Path) -> bool:
        """Incremental update for a single file. Returns True if changed."""

    def query(self) -> CodeGraphQuery:
        """Return query interface."""

class CodeGraphQuery:
    def find_symbol(self, name: str) -> list[SymbolInfo]: ...
    def find_callers(self, function_name: str) -> list[SymbolInfo]: ...
    def find_callees(self, function_name: str) -> list[SymbolInfo]: ...
    def find_dependencies(self, module_path: str) -> list[DependencyEdge]: ...
    def get_call_graph(self, entry_point: str, max_depth: int = 3) -> dict: ...
    def find_similar(self, signature: str) -> list[SymbolInfo]: ...
    def get_stats(self) -> dict: ...
```

### 1.5 集成点

- **Worker**: `worker.py` — Worker 执行前调用 `graph.query().find_symbol()` 获取上下文
- **DualLayerContextManager**: 项目级上下文从图谱获取
- **Dispatcher**: 可选参数 `code_graph: CodeKnowledgeGraph | None = None`

## 3. V39-03: YagniChecker

### 3.1 接口定义

```python
class YagniResult:
    verdict: str  # SKIP|USE_STDLIB|USE_DEPENDENCY|ONE_LINER|MINIMAL|NECESSARY
    reason: str
    upgrade_path: str  # 如何升级到完整实现
    shortcut_marker: str  # "shortcut: <reason>"

class YagniChecker:
    NEVER_SKIP_PATTERNS = [
        r"(?i)(validate|check|verify|sanitize|escape|escape_html)",
        r"(?i)(error|exception|catch|handle|fallback)",
        r"(?i)(permission|auth|rbac|access)",
        r"(?i)(backup|save|persist|commit)",
    ]

    def check(self, task: MicroTask) -> YagniResult: ...
```

### 3.2 集成点

- MicroTaskPlanner.plan() → 生成微任务后 → 对每个微任务运行 YagniChecker
- SKIP 的微任务标记为 skipped，不执行

## 4. V39-04: PromptDials

### 4.1 接口定义

```python
@dataclass
class PromptDials:
    verbosity: int = 3      # 1-5
    creativity: int = 3     # 1-5
    risk_tolerance: int = 3 # 1-5

    @classmethod
    def from_variant(cls, variant: str) -> "PromptDials":
        """Backward compat: variant='concise' → (1,3,3), 'balanced' → (3,3,3), 'detailed' → (5,3,3)"""
```

### 4.2 PromptAssembler 集成

- `assemble()` 接受可选 `dials: PromptDials`
- 如果传了 `variant` 但没传 `dials`，用 `from_variant()` 转换
- 根据 dials 值注入 prompt 片段

## 6. V39-06: RBAC/Audit 集成

### 6.1 RBAC 接口 (采纳 S2 建议: RBAC0)

```python
class DispatchRBAC:
    def check_dispatch_permission(
        self, user_id: str, roles: list[str], mode: str
    ) -> tuple[bool, str]:
        """Check if user can dispatch with given roles/mode. Returns (allowed, reason)."""
```

### 6.2 Audit Logger (采纳 S3 建议: append-only + SHA-256)

```python
class DispatchAuditLogger:
    def log_dispatch_start(self, user_id: str, task: str, roles: list[str]): ...
    def log_dispatch_end(self, user_id: str, success: bool, duration: float): ...
    def log_permission_denied(self, user_id: str, reason: str): ...
    # Storage: append-only SQLite + SHA-256 chain hash
```

### 6.3 集成点

- Dispatcher.__init__() 接受 `rbac: DispatchRBAC | None` 和 `audit_logger: DispatchAuditLogger | None`
- dispatch() 开头检查 RBAC → 记录 audit start → 执行 → 记录 audit end
